Independent. Bare metal Linux. Go programming.

I believe modern systems are deliberately over-abstracted — hiding the machine behind unnecessary layers. To truly own a system, you have to understand every layer and its purpose. In the spirit of the Unix philosophy, I prefer tools that do one job well and directly — where a terminal will work, I want no wrappers, no GUIs and no abstractions I don't control.

I use ffmpeg, not OBS. I run SLMs bare metal on llama.cpp directly — not through Ollama or Jan. The Go programming language fits this mindset well: small static binaries, no external dependencies, built once and able to run anywhere Linux runs.

I build Linux utilities in Go and Bash. Diagnostic tools, hardware probes, system automation, etc. I'm a FOSS and Right to Repair advocate and active in hardware restoration and returning aging or broken machines and devices back to productive use.

Stack: CrunchBang++ / Openbox / tint2 — Go, Bash — static binaries, bare metal, no unnecessary dependencies.
